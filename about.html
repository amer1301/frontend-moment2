Information om webbplatsen

Denna webbplats är skapad för att hantera arbetserfarenheter i ett CV-format. Användaren kan lägga till, läsa, uppdatera och ta bort information om sina tidigare arbeten med så kallad CRUD-funktionalitet (Create, Read, Update, Delete).


Webbplatsen är uppdelad i två delar, en backend och en frontend. Frontend är byggd med hjälp av HTML, JavaScript och CSS. Backend (webbtjänsten) är en REST API byggd i Node.js med express och MySQL som hanterar databasfrågor och kommunicerar med frontend via HTTP. Servern exponerar flera API-endpoints via /api/workexperience/ vilket möjliggör kommunikationen mellan frontend-delen och backend för att kunna uppdatera informationen dynamiskt. CORS användes för att tillåta frontend-appen (som körs från en annan domän) att göra anrop till backend-servern. Frontend-delen publicerades via Netlify och backend vid Render.


Denna uppgift har klargjort hur frontend och backend fungerar tillsammans och jag vart även glad över att lära mig hur CORS fungerar med tanke på att detta i tidigare moment förstörde de initiala idéer jag hade för uppgiften. Uppgiften gav mig en djupare förståelse för hur frontend och backend samverkar via API:er och hittills skulle jag nog säga att det varit den roligaste kursen.
